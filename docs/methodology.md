# Methodology

This document describes the methodology followed in the satellite image classification project.

The project is structured as a sequence of experimental stages designed to progressively improve model performance, interpretability, and robustness.

## Stage 1 – Baseline CNN model
A baseline convolutional neural network (CNN) was implemented and evaluated using the **UC Merced Land Use Dataset**.

This stage included:
- Dataset loading and preprocessing
- Stratified train/validation/test split
- Definition of a baseline CNN architecture
- Model training with data augmentation
- Performance evaluation using accuracy, confusion matrix, and class-wise analysis

The baseline model serves as the reference point for all subsequent experiments.

## Stage 2 – Training improvements
The baseline CNN was progressively refined through a set of controlled modifications.

This stage was divided into three iterative configurations:
- **B1:** deeper architecture and architectural regularization
- **B2:** regularization and dropout tuning
- **B3:** refinement of training dynamics through callback adjustments

The purpose of this stage was to improve generalization performance while maintaining methodological consistency with the baseline setup.

## Stage 3 – Model explainability (Saliency Maps)
To analyze the internal behavior of the trained CNN, gradient-based saliency maps were generated using the final improved model.

This stage included:
- Computation of gradients with respect to the predicted class
- Saliency map generation using `tf.GradientTape()`
- Channel reduction and normalization
- Gamma enhancement for improved visualization
- Qualitative inspection of representative test images from different classes

This stage provides interpretability evidence by highlighting the image regions that most influence the model predictions.

## Stage 4 – Robustness analysis under image degradation
The robustness of the trained CNN was evaluated under different image degradation scenarios.

Three degradation families were considered:

### 4.1 Resolution degradation
Additional datasets were generated by reducing the original image resolution to:
- 128 × 128
- 64 × 64
- 32 × 32

This experiment evaluates how classification performance changes as spatial detail decreases.

### 4.2 AWGN noise degradation
Additional datasets were generated by injecting **Additive White Gaussian Noise (AWGN)** at different SNR levels:
- 20 dB
- 10 dB
- 5 dB

This experiment evaluates the sensitivity of the model to increasing noise levels.

### 4.3 Kernel-based blur degradation
Additional degraded datasets were generated using normalized average blur kernels of size:
- 2 × 2
- 4 × 4
- 8 × 8
- 16 × 16

This experiment evaluates the impact of spatial smoothing and local detail loss on classification performance.

## Stage 5 – Future extensions
Future work may include:
- more robust CNN architectures
- transfer learning with pretrained backbones
- comparative experiments with additional degradation models
- broader robustness analysis under combined degradations

## General workflow
Across all stages, the methodology follows the same general logic:
1. Prepare or generate the corresponding dataset
2. Apply the selected model or trained reference model
3. Evaluate quantitative performance
4. Analyze qualitative behavior when relevant
5. Compare results against the baseline and improved configurations
